控制多媒体播放

方法：

  • load() 加载媒体内容
  • play() 开始播放

  • pause() 暂停播放

    属性：

  • playbackRate 播放速度（0-1慢速 1正常速度 >1快速播放）

  • currentTime 播放进度
  • volume 音量

  • muted 静音

    查询多媒体状态

  • paused 暂停

  • seeking 跳转
  • ended 播放完成
  • duration 媒体时长

  • initialTime 媒体开始时间
    多媒体相关事件

  • loadstart 开始请求媒内容

  • loadmetadata 媒体元数据已经加载完成
  • canplay 加载了一些内容，可以开始播放
  • play 调用play()，或设置了autoplay
  • waiting 缓冲数据不够，播放暂停
  • playing 正在播放

事件流

w3c标准的事件对ie低版本没有捕获过程
有些事件不包括全部过程页面的load事件没有冒泡

  • capture phase
    • 事件捕获
  • target phase
    • 事件触发
  • bubble phase
    • 事件冒泡

事件注册与触发

  • 事件注册

  • 取消时间注册

  • 事件触发

事件注册

eventTarget.addEventListener(type,listener[,useCapture])

其中useCapture表示是否启用事件捕获，默认为false。即使用时间冒泡。

  • DOM 0级事件
    • 只能注册一个回调函数
  • DOM 2级事件
    • 可以注册多个回调函数

取消事件注册

eventTarget.removeEventListener(type,listener[,useCapture])

其中useCapture表示是否启用事件捕获，默认为false。即使用时间冒泡。

事件触发

eventTarget.dispatchEvent(type)

浏览器兼容性(IE6,7,8)

  • 事件注册于取消
    • attchEvent/detachEvent
  • 事件触发
    • fireEvent(e)
  • no capture

事件对象

事件的回调函数中传入的形参就是该事件的事件对象

事件对象的属性和方法

  • 属性
    • type
    • target(srcElement)
    • currentTarget
  • 方法
    • stopPropagation
    • preventDefault
    • stopImmediatePropagation

阻止事件传播

  • event.stopPropagation() //（W3C）
    • 阻止事件冒泡
  • event.cancelBubble = true //（IE）
    • 在IE低版本中阻止事件的冒泡
  • event.stopImmediatePropagation() //（W3C）
    • 阻止事件冒泡，同时将阻止后续注册的事件触发

阻止默认行为

  • event.preventDefault() //（W3C）
  • event.returnValue = false //（IE）

事件分类

其中继承的事件可以拥有父类的事件属性和方法

MouseEvent

事件

对比mouseover和mouseenter

  • mouseover在鼠标进入其嵌套元素时会触发
  • mouseenter在鼠标进入其嵌套元素时不会触发

对比mouseout和mouseleave

  • mouseleave在鼠标离开元素及其其嵌套元素时才会触发
  • mouseout在鼠标进入元素的嵌套元素时会触发

    属性

  • clientX,clientY
  • screenX,screenY
  • ctrlKey,shiftKey,altKey,metaKey
  • button(0,1,2)

    MouseEvent顺序

WheelEvent

继承自MouseEvent,MouseEvent的全部属性和方法全都继承

  • deltaMode
  • deltaX
  • deltaY
  • deltaZ

FocusEvent

    • 当元素失去焦点时，获得焦点的元素是relatedTarget
    • 当元素获得焦点时，失去焦点的元素是relatedTarget

样式操作

CSS→DOM

更新样式

  • element.style：增加了内嵌样式表，每一次设置属性都只能一句句设置
    element.style
  • element.style.cssText：增加了内嵌样式表，只需要用一条语句设置，具有很强的扩展性，且使用我们熟悉的CSS
    element.style.cssText

上述这两种方法修改CSS都会导致一个问题，就是样式是混在逻辑当中，那每一次我们修改CSS，都需要到JS修改，这肯定不是我们想要的

  • element.className：即通过更新Class给元素增加类名，使用已写好的CSS更新样式
    但是当我们需要一次性修改大量样式时，编写大量的class会增加大量的类名，这并不是很好的办法，这个时候，我们就可以更换另外一份CSS样式表

  • href：XXX.css

    获取样式

    element.style：对应元素内嵌样式表，获得的并不是实际的样式表
    window.getComputedStyle(element)：可以获取实际样式表
