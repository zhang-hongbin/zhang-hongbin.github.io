事件流

w3c标准的事件对ie低版本没有捕获过程
有些事件不包括全部过程页面的load事件没有冒泡

  • capture phase
    • 事件捕获
  • target phase
    • 事件触发
  • bubble phase
    • 事件冒泡

事件注册与触发

  • 事件注册

  • 取消时间注册

  • 事件触发

事件注册

eventTarget.addEventListener(type,listener[,useCapture])

其中useCapture表示是否启用事件捕获，默认为false。即使用时间冒泡。

  • DOM 0级事件
    • 只能注册一个回调函数
  • DOM 2级事件
    • 可以注册多个回调函数

取消事件注册

eventTarget.removeEventListener(type,listener[,useCapture])

其中useCapture表示是否启用事件捕获，默认为false。即使用时间冒泡。

事件触发

eventTarget.dispatchEvent(type)

浏览器兼容性(IE6,7,8)

  • 事件注册于取消
    • attchEvent/detachEvent
  • 事件触发
    • fireEvent(e)
  • no capture

事件对象

事件的回调函数中传入的形参就是该事件的事件对象

事件对象的属性和方法

  • 属性
    • type
    • target(srcElement)
    • currentTarget
  • 方法
    • stopPropagation
    • preventDefault
    • stopImmediatePropagation

阻止事件传播

  • event.stopPropagation() //（W3C）
    • 阻止事件冒泡
  • event.cancelBubble = true //（IE）
    • 在IE低版本中阻止事件的冒泡
  • event.stopImmediatePropagation() //（W3C）
    • 阻止事件冒泡，同时将阻止后续注册的事件触发

阻止默认行为

  • event.preventDefault() //（W3C）
  • event.returnValue = false //（IE）

事件分类

其中继承的事件可以拥有父类的事件属性和方法

MouseEvent

事件

对比mouseover和mouseenter

  • mouseover在鼠标进入其嵌套元素时会触发
  • mouseenter在鼠标进入其嵌套元素时不会触发

对比mouseout和mouseleave

  • mouseleave在鼠标离开元素及其其嵌套元素时才会触发
  • mouseout在鼠标进入元素的嵌套元素时会触发

    属性

  • clientX,clientY
  • screenX,screenY
  • ctrlKey,shiftKey,altKey,metaKey
  • button(0,1,2)

    MouseEvent顺序

WheelEvent

继承自MouseEvent,MouseEvent的全部属性和方法全都继承

  • deltaMode
  • deltaX
  • deltaY
  • deltaZ

FocusEvent

属性

  • relatedTarget
    • 当元素失去焦点时，获得焦点的元素是relatedTarget
    • 当元素获得焦点时，失去焦点的元素是relatedTarget
